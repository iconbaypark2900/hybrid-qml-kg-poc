# Quantum ML Implementation Execution Plan

## Alignment

Implements `docs/QUANTUM_ML_IMPLEMENTATION_PLAN.md` as-is, while:

- Preserving `experiments/` (research utilities; excluded from CI/build)
- Keeping `scripts/rbf_svc_fixed.py` active; extracting its strong pieces into shared utilities used by `scripts/run_pipeline.py`
- Using `scripts/run_pipeline.py` as the canonical E2E entrypoint

## Immediate Quick Wins (Week 1-2)

- VQC loss tracking and plotting
- Edit `quantum_layer/qml_model.py`: add optimizer registry (COBYLA, SPSA, NFT, GradientDescent/Adam) and callback hook to capture per-iteration loss
- Edit `quantum_layer/qml_trainer.py`: pluggable callbacks; persist `losses` to `results/qml_vqc_loss_{stamp}.json`; optional PNG plot if `matplotlib` available
- Classical regularization sweep
- Edit `classical_baseline/train_baseline.py`: function `regularization_path(X_train, y_train, X_test, y_test, C_values)`; save CSV/plot under `results/`
- Create `scripts/regularization_path.py` CLI wrapper
- Embedding quality checks
- Edit `kg_layer/kg_embedder.py`: utilities `get_embedding(entity_id)`, `get_all_embeddings(entity_type)`, `compute_similarity_stats(drug_classes)` and `tsne_visualize()`; save figures/metrics to `results/embeddings/`
- Multi-seed validation
- Create `scripts/multi_seed_experiment.py` to run the full pipeline across seeds and emit JSON summary + Wilcoxon test

## Phase 1: VQC Optimization (Week 3-4)

- Optimizer comparison
- Create `scripts/compare_optimizers.py` using `qml_model.QMLLinkPredictor` optimizer registry and callback logging
- Ansatz architecture search
- Create `scripts/ansatz_search.py` evaluating `RealAmplitudes`, `EfficientSU2`, `TwoLocal` and reps ablation; log params, depth, PR-AUC
- Edit `quantum_layer/qml_model.py`: extend `_build_ansatz` to support `TwoLocal`
- Feature engineering for quantum
- Create `kg_layer/feature_engineering.py` with encodings: diff, hadamard, concat, polynomial; normalization strategies; expose `make_qml_features(h, t, strategy, qml_dim)`
- Edit `kg_layer/kg_embedder.py`: call feature strategies for QML features
- Hyperparameter search
- Create `scripts/hyperparameter_search.py` with `ParameterGrid` over `ansatz_reps`, `feature_map_reps`, `optimizer`, `max_iter`; 5-fold CV PR-AUC

## Phase 2: Classical Baseline Improvement (Week 5-6)

- Cross-validation and CI-ready evaluation
- Create `scripts/nested_cv.py` for unbiased nested CV on classical models; report mean±std and 95% CI
- Edit `classical_baseline/train_baseline.py`: expose `train_rbf_svc_cv(...)` returning oof metrics, best params, and test metrics (parity with `rbf_svc_fixed.py`)
- Model complexity & learning curves
- Create `scripts/model_comparison.py` to evaluate Logistic, Ridge, SVM-Linear/RBF, RF, GBM, MLP via CV PR-AUC
- Create `scripts/learning_curves.py` to generate training-size curves

## Phase 3: Data Quality & Scale (Week 7-8)

- Embedding validation suite
- Create `scripts/embedding_validation.py` to run within/between similarity, t-SNE, and label correlation
- Embedding algorithm comparison (PyKEEN)
- Create `scripts/embedding_comparison.py` to run TransE/DistMult/ComplEx/RotatE/TuckER; gate behind optional `pykeen` dependency
- Hard negative mining
- Edit `kg_layer/kg_loader.py`: hooks for `get_hard_negatives_similarity` and `get_hard_negatives_adversarial` (optional)
- Create `scripts/hard_negatives_experiment.py` to compare strategies
- Scaling study
- Create `scripts/scaling_study.py` to run N = [300, 500, 1000, 2000, 5000] and log PR-AUC, time, params

## Phase 4: Scalability Validation (Week 9-10)

- Empirical runtime measurement
- Create `benchmarking/empirical_scaling.py` measuring train/inference times for classical and quantum; fit/quadratic vs log models; plot & crossover point
- Complexity profiling
- Create `benchmarking/profile_pipeline.py` using `cProfile` and top-20 cumulative report; optional memory profiling
- Statistical significance tests
- Create `scripts/stat_tests.py` to compute bootstrap CIs, Wilcoxon/t-tests, Cohen's d from saved results

## Phase 5: Hardware Validation & Docs (Week 11-12)

- IBM Quantum execution
- Ensure `quantum_layer/quantum_executor.py` selects backends; add resilience levels via `Options` when available; run simulator vs hardware comparison script `scripts/backend_compare.py`
- Error mitigation study
- Extend `backend_compare.py` to sweep `resilience_level` and shots; record PR-AUC/time/cost
- Documentation
- Add `docs/THEORY.md`, `docs/ARCHITECTURE.md`, `docs/TUTORIAL.md`, `docs/WHEN_TO_USE_QUANTUM.md`; cross-link from `README.md`

## Pipeline Unification & Parity

- Canonical E2E entry: `scripts/run_pipeline.py` with flags for classical/quantum paths, seeds, and configs
- Refactor `scripts/rbf_svc_fixed.py` into a thin wrapper calling `classical_baseline/train_baseline.py` and shared feature utilities; keep file to avoid breaking workflows
- Add `scripts/e2e_smoke.py` (small N, <3 minutes) for CI
- Store all metrics as JSON under `results/` with unified schema; generate brief text summary

## Notes

- Optional deps (hardware, PyKEEN) are gated; core pipeline runs with scikit-learn + qiskit
- No deletion of `experiments/`; add a short README and mark as non-entry for CI
